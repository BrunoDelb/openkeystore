WebCrypto++ Payment Demo Messages

This document shows the message exchanges of the WebCrypto++ Payment Demo. It is supposed to be read together with the state diagram. Notation: Payee = Merchant, Payment Provider = Bank.
Note that there are three independent PKIs involved: The Payment Network PKI, The Payee Network PKI, The local Bank client PKI. In addition there are also Payment Provider-specific encryption keys.
State #1: When the Payment Application (IFRAME) has been properly loaded, it signals [using postMessage()] to the Payee holding window that it is ready.
